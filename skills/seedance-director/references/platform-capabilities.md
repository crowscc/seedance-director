# 即梦 Seedance 2.0 平台能力与参考

> 即梦平台全部 10 种生成能力的模式、示例和技术参数。当需要选择生成方式或编写提示词时查阅此文件。

## Seedance 2.0 平台能力速查

### 纯文本生成（Text-to-Video）

无需素材，最基础的能力。

**模式**：`(主体) + (动作序列) + (环境/光影) + (镜头语言) + (风格)`

```
一位身穿黑色风衣的男人站在雨中的霓虹街头，缓缓抬头望向天空，
雨水顺着脸颊滑落。镜头从中景缓推至面部特写。
赛博朋克风格，霓虹灯光映射在湿润的路面上，体积光穿透雨幕。
电影质感，宽银幕构图，浅景深。
```

### 一致性控制（角色/场景一致）

通过@引用图片锁定角色外观和场景风格。多角色同框时，每个角色绑定各自的参考图（@图片N），平台自动区分并保持一致性。不要假设"多角色场景一致性差" — 只要参考图到位，角色外观在跨镜头和跨段落间均可稳定保持。

**模式**：`[角色]@图片N + [动作剧情] + [场景]@图片N + [运镜]`

```
@图片1 中的女孩走进 @图片2 中的咖啡馆，坐到靠窗的位置，
拿起菜单翻看，微微一笑。
镜头从门口跟随她移动到座位，自然光从窗外照入，暖色调。
```

### 运镜复刻（Motion Reference）

借鉴参考视频的运镜方式，用在新的主体和场景上。

**模式**：`参考@视频1的[运镜] + [主体]@图片N + [场景]`

```
参考 @视频1 的运镜方式和节奏，
@图片1 中的武侠角色在竹林中拔剑出鞘，衣袂飘动。
保持 @视频1 的镜头运动轨迹，替换为中国风水墨画面质感。
```

### 特效复刻（Effect Reference）

借鉴参考视频的特效风格，用在新的元素上。

**模式**：`参考@视频1的[特效] + 将[元素]替换为@图片N`

```
参考 @视频1 中的粒子爆炸特效和光影变化，
将爆炸中心的物体替换为 @图片1 中的产品，
产品从粒子中凝聚成形，360度旋转展示，金色光芒四射。
```

### 剧情补全（Story Completion）

基于图片或简单描述，让 AI 补全中间的剧情动画。

**模式**：`[分镜/图片描述] + [演绎方式] + [音效台词]`

```
@图片1 是起始画面（女孩站在教室门口），@图片2 是结束画面（女孩坐在座位上微笑）。
补全中间过程：女孩推门进入教室，走过课桌之间的过道，在靠窗座位坐下，
转头望向窗外阳光，露出微笑。自然光线，校园青春风格。
```

### 视频延长（Video Extension）

在已有视频基础上续写新内容，**适合 2 段拼接（≤30s）的续写能力**。

**模式**：`将@视频1延长[X]s + [新增内容]`

**关键**：时长设置选**新增部分的时长**（不是总时长），新内容起始画面必须与原视频结尾自然衔接。

**适用范围**：建议仅用于 2 段拼接（≤30s）。3 段及以上的视频，视频延长会导致错误传递、风格漂移和控制力递减，应改用「独立生成 + 首帧衔接」策略。

```
将 @视频1 延长10秒。
接续画面：角色从桌前站起，走向窗边，推开窗户，
镜头跟随移动后切换为窗外的城市远景，夕阳西下，暖色调渐浓。
保持与原视频一致的画面风格和色调。
```

### 声音控制（Voice & Audio）

控制视频中的台词、音色和音效。平台完全支持中文对白与口型同步 — 角色说话时嘴型自动匹配台词内容，无需额外处理。短剧、对白类视频应直接使用台词，不必降级为旁白。

**模式**：`[画面] + 音色参考@视频1 + "台词内容"`

```
一位中年教授站在黑板前，指着公式讲解。
音色参考 @视频1 中的男声，语调沉稳，语速中等。
台词："这个公式告诉我们，宇宙的膨胀速度正在加快。"
教室环境音，粉笔书写声。
```

### 一镜到底（Long Take）

多张图片串联成连续长镜头，中间不切镜。

**模式**：`一镜到底 + @图片1234 + [连续场景] + 不切镜头`

```
一镜到底，不切镜头，镜头连续平滑移动：
从 @图片1 的街道入口开始 → 经过 @图片2 的市集摊位 →
穿过 @图片3 的拱门 → 最终到达 @图片4 的广场喷泉。
手持跟拍质感，自然光，人群熙攘，生活气息。
```

### 视频编辑（Video Editing）

在已有视频基础上修改特定元素。

**模式**：`将@视频1中的[A]换成@图片1 + [修改说明]`

```
将 @视频1 中桌上的红色杯子替换为 @图片1 中的蓝色陶瓷杯。
保持视频其他元素不变，替换后的杯子光影与环境一致。
```

### 音乐卡点（Music Sync）

画面切换跟随音乐节拍。

**模式**：`@图片1234 + 参考@视频1节奏 + [风格]`

```
将 @图片1、@图片2、@图片3、@图片4 按照 @音频1 的节拍进行切换。
每个鼓点切换一张图片，图片之间使用动态转场。
时尚杂志风格，高饱和色彩，快节奏剪辑感。
```

## 技术参数与限制

| 维度 | 规格 |
|------|------|
| 图片 | jpeg/png/webp/bmp/tiff/gif，最多 9 张，单张 < 30MB |
| 视频 | mp4/mov，最多 3 个，2-15秒，单个 < 50MB，480p-720p |
| 音频 | mp3/wav，最多 3 个，每段 <= 15秒，单个 < 15MB |
| 混合上限 | 所有文件合计最多 12 个 |
| 生成时长 | 4-15 秒（单次生成） |
| 宽高比 | 16:9 / 9:16 / 1:1（2.35:1宽银幕需后期裁切16:9画面） |
| 写实人脸 | 不支持上传含清晰真人脸部的素材（平台安全策略） |

**注意**：视频延长时时长设为新增部分时长；超 15 秒必须多段拼接；上传顺序决定@引用编号。多段视频建议先生成角色三视图和场景概念图作为一致性锚点。

## @引用系统规范

| 类型 | 范围 | 格式 |
|------|------|------|
| 图片 | 1-9 | `@图片1` ~ `@图片9` |
| 视频 | 1-3 | `@视频1` ~ `@视频3` |
| 音频 | 1-3 | `@音频1` ~ `@音频3` |

**规范**：
1. **必须中文名称**：写 `@图片1` 而不是 `@image1`
2. **必须标注用途**：每个@引用后面说明素材在画面中的作用
3. **区分「参考」和「编辑」**：
   - 参考：借鉴风格/运镜/特效，生成新内容 → "参考 @视频1 的运镜"
   - 编辑：直接在原素材上修改 → "将 @视频1 中的 A 换成 B"

```
正确：@图片1 作为主角的形象参考，@图片2 作为场景背景
正确：参考 @视频1 的运镜方式，应用到新画面中
错误：@image1（不要用英文）
错误：@图片1（没有说明用途）
```
